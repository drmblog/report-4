Introduction:

For the purpose of last assignment the graph was created for a semantic network based on the relevant news data.

Process

Data was gather with Lexis Nexis power search. The search query was «North Korea» and it was limited to major world publications which occurred in the period of last three months. The full dataset included 188 unique articles. Data was then processed in Context using Network Construction function specifically designed for Lexis Nexis data. After that, the map was created using the VOS viewer software with the help of full counting method. Each viz was slightly modified using internal VOS viewer setting, which are quite limited.
An entity-based network was also created using Context software for the same dataset. The distance parameter was set to 25. This step produced several semantic network files, one of which was ready to be directly loaded in Gephi software as well as yEd graph editor software. Force Atlas method was used in order to reorganize the graph. After that modularity test was executed, as well as some others analysis to measure nodes centrality. The graph was colorized depending on the modularity attribute of nodes. After that the spread distance was changed several time and the nodes labels were turned to be shown on the graph. Minor tweaks enabled the graph to become a bit more readable.

It was possible to identify several «small worlds» on the graph, one of which was strongly linked to the Middle East context.

Unfortunately, all possible stable releases (including new 0.9) of Gephi could not take me further than those steps. Each time I tried to make any changes I got crashes, freezes or simple interface rendering issues, which did not even allow me to turn back the labels off for the existing graph. I only once was lucky enough to get to the «data» screen, which freezed the time after I tried to make a simple click.
